# Face Recognition Attendance System

## Files
```
face_attendance_system/
├── encode_faces.py        ← Step 1: build encodings from dataset
├── recognize_realtime.py  ← Step 2: live recognition + attendance
├── attendance.py          ← CSV logger module (imported automatically)
├── requirements.txt
├── dataset/               ← YOU create this
│   ├── Alice/
│   │   ├── 1.jpg
│   │   └── 2.jpg
│   └── Bob/
│       └── 1.jpg
├── encodings.pkl          ← auto-generated by Step 1
└── attendance.csv         ← auto-generated by Step 2
```

## Setup
```bash
pip install cmake dlib face-recognition opencv-python imutils numpy
```

## Usage

### Step 1 — Build face encodings (run once per dataset update)
```bash
python encode_faces.py
```

### Step 2 — Start live attendance
```bash
python recognize_realtime.py
```
- Press **Q** to quit
- Press **S** to print today's session list in terminal

## CSV Output (attendance.csv)

Name,Date,Time

## Key Tuning Parameters (in recognize_realtime.py)
| Parameter | Default | Effect |
|---|---|---|
| `TOLERANCE` | 0.45 | Lower = stricter matching |
| `PROCESS_EVERY_N` | 3 | Higher = faster, less accurate |
| `DETECTION_METHOD` | "hog" | "cnn" = more accurate (needs GPU) |

## Tips for Better Accuracy
- Use 5–10 images per student with varied lighting/angles
- Ensure images are clear (no blur, face fully visible)
- Lower TOLERANCE to 0.4 to reduce false positives
- Switch DETECTION_METHOD to "cnn" if you have a GPU
